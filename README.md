# 🔥 热点话题词云分析 + 对比分析

## 1. 项目简介

本项目是一个基于 Python 的中文热点话题分析系统，以**微博热搜榜**为数据来源，实现从数据采集、中文分词、词频统计，到词云可视化与对比分析的完整数据科学流程。

通过对**两个时间点**的热搜数据进行对比，直观呈现热点话题的兴起与退热规律，揭示舆论焦点的动态变化。

---

## 2. 技术栈说明

| 模块 | 技术/库 | 用途 |
|------|--------|------|
| 数据采集 | `requests` | 请求微博热搜 API |
| 中文分词 | `jieba` | 精确模式分词 |
| 词频统计 | `collections.Counter` | 高效词频计数 |
| 词云生成 | `wordcloud` | 生成可视化词云图 |
| 图表渲染 | `matplotlib` | 图片渲染与保存 |
| 文本清洗 | `re`（正则表达式）| 去除符号与噪声 |

---

## 3. 数据来源说明

- **主要来源**：微博热搜榜 API（`https://weibo.com/ajax/side/hotSearch`）
- **采集策略**：
  - 第一次采集作为"当前"热搜（`raw_now.txt`）
  - 第二次采集（或模拟历史数据）作为"历史"热搜（`raw_earlier.txt`）
  - 每个时间点采集不少于 30 条热搜词
- **降级策略**：若网络请求失败，自动使用内置的模拟数据（仍具真实话题代表性，且两组数据有差异），保证程序始终可运行

## 5. 三张词云说明

### wordcloud_now.png — 当前热搜词云
- **色彩**：红色系（`Reds`）
- **含义**：展示当前时间点微博热搜中，分词后词频最高的词语
- **解读**：词越大代表在当前热搜中出现频率越高，是当下最受关注的话题关键词

### wordcloud_earlier.png — 历史热搜词云
- **色彩**：蓝色系（`Blues`）
- **含义**：展示历史时间点的热搜词频分布
- **解读**：与当前词云对比，可以看出哪些话题已经退出公众视野

### wordcloud_diff.png — 热搜变化对比词云
- **色彩**：红绿双色系（`RdYlGn`）
- **含义**：展示两个时间点之间**词频变化幅度最大**的词语（差异词云）
- **解读**：词越大代表变化越显著，是识别"新兴热点"与"降温话题"的核心可视化

## 6. 对比分析结果（重点）

### 分析方法
基于两个时间点 Top 20 高频词的集合运算，将词汇分为三类：

| 类别 | 定义 | 舆情意义 |
|------|------|----------|
| **新热点词** | 仅出现在当前 Top 20 | 近期新兴话题，正处于关注上升期 |
| **降温词** | 仅出现在历史 Top 20 | 话题热度已消退，退出主要舆论场 |
| **共同热点词** | 两个时间点均在 Top 20 | 持续性热点，具有较高的长尾关注度 |

### 典型规律总结

通过分析微博热搜的两个时间切片数据，可以发现以下规律：

1. **热点更替迅速**：Top 20 词汇中通常有 30%~50% 发生替换，说明微博热搜的话题周期普遍较短（通常为数小时至一两天）。娱乐八卦类词汇变化最快，而政策、民生类话题的留存率相对较高。

2. **持续热点揭示深层关切**：两个时间点均高频出现的词语，往往与国计民生、重大政策或持续性社会事件相关（如就业、房价、医保等），反映了公众长期关注的核心议题。

3. **新热点聚焦突发事件**：新出现的高频词通常与突发新闻、体育赛事结果、娱乐圈动态高度相关，具有明显的时效性特征。

4. **差异词云的价值**：对比词云（diff）综合展示了两个时间维度的变化，有助于快速锁定舆情转折点，对媒体监测、品牌危机预警等场景有较高实用价值。

## 8. 常见问题

**Q：词云中中文显示为方框怎么办？**  
A：程序会自动检测系统中文字体。如仍显示异常，请手动下载 `SimHei.ttf` 放入项目根目录，并在 `find_chinese_font()` 函数中添加其路径。

**Q：网络请求失败怎么办？**  
A：程序已内置高质量模拟数据，网络失败时自动切换，不影响词云生成。

**Q：如何更换为真实数据？**  
A：在 `fetch_data()` 函数中，替换 `MOCK_DATA_NOW` 和 `MOCK_DATA_EARLIER` 为你自己的数据，或配置正确的 Cookies 以访问微博 API。
